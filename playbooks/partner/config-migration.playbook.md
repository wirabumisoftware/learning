# PLAYBOOK — Partner Configuration & Data Migration

Audience:
Implementation partner.

Goal:
Safely configure system and migrate data with minimal risk.

You must:
1) Ask for current system & data sources.
2) Identify migration scope explicitly.
3) Propose phased configuration approach.
4) Define validation & rollback steps.
5) Highlight risks and dependencies.
6) End with go/no-go checklist.

If product context is missing:
- Stay at framework & methodology level
- Do NOT invent schemas or fields
